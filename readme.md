# 计算机网络实验一：数据链路层滑动窗口协议设计与实现

## 项目简介

本仓库用于《计算机网络实验一：数据链路层滑动窗口协议设计与实现》的课程实验开发、测试与文档整理。
项目基于给定仿真环境，在有误码信道条件下实现数据链路层可靠双工通信，并完成联调、性能测试与实验报告整理。

本仓库面向 2 人学生小组协作维护，代码实现、测试记录、实验文档与报告素材统一存放于此，便于开发、复现与提交。

------

## 实验目标

本实验的核心任务是在给定的仿真环境下，设计并实现一个数据链路层滑动窗口协议，使两个站点能够在存在误码的信道上进行可靠双工通信。
围绕该目标，仓库中的工作主要包括：

- 设计协议的基本机制，包括帧格式、发送与接收流程、确认与重传逻辑
- 在 `datalink.c` 中实现数据链路层核心协议代码
- 在仿真环境中完成正确性验证，包括无误码和有误码场景下的运行测试
- 观察并分析协议在不同测试条件下的行为与性能
- 整理联调记录、性能测试结果和实验报告素材

当前小组实现方案以**选择重传协议（Selective Repeat, SR）**为主。

## 开发环境说明

本项目默认面向课程实验提供的开发环境进行维护，推荐使用以下环境之一：

- **Windows**
	- Visual Studio 工程环境
	- VS Code + Visual Studio 编译工具链
- **Linux**
	- GCC / Makefile 环境（如课程提供 Linux 版本工程）

实验所依赖的基础文件主要包括：

- `datalink.c`：小组主要实现的核心协议代码
- `protocol.c / protocol.h`：实验环境提供的接口与仿真支持
- `crc32.c`：CRC 相关支持
- `lprintf.c / lprintf.h`：日志输出支持
- `getopt.c / getopt.h`：命令行参数处理支持

## 编译与运行方式

请根据实际使用的 **Windows 工程** 或 **Linux Makefile 环境** 进行编译运行。

### Windows

- 使用仓库中的 Visual Studio 工程文件打开项目
- 在工程中完成编译
- 按实验要求分别启动 A 站与 B 站进程
- 运行时关注日志输出、双站连接情况与是否出现错误中止

### Linux

- 如使用课程提供的 Linux 环境，请根据对应 Makefile 进行编译
- 编译完成后分别启动两个站点进行联调测试

### 运行建议

- 优先先做基础场景联调，再逐步进入有误码、长时间运行和性能测试
- 每轮测试建议保留：
	- 使用的代码版本
	- 启动参数
	- 测试现象
	- 日志片段
	- 结论说明

## 协作说明

本仓库面向 2 人课程实验小组协作维护，采用“**一人主责核心实现，一人主责测试验证与性能分析，关键逻辑交叉评审**”的方式推进。

简要约定如下：

- `datalink.c` 由核心实现负责人主维护，避免长期无边界并行修改
- 测试、日志整理、性能分析与报告素材由测试负责人主整理
- 关键逻辑、关键测试结论、报告中的核心分析内容需要互相确认

详细协作规则见：

- `docs/01-协作方案.md`
- `docs/02-两人分工文档.md`

## 注意事项

- 本实验核心实现主要集中在 `datalink.c`，修改时应注意控制边界，避免多人同时长期改同一段逻辑
- `protocol.c / protocol.h / crc32.c / lprintf.c` 等老师提供文件原则上不随意修改
- 每次阶段性修改后，建议先做基础编译与双站启动验证
- 测试不只记录成功结果，失败现象、异常日志和问题定位过程同样需要保留
- 性能测试与实验报告整理不要留到最后统一处理，建议随开发进度同步记录
