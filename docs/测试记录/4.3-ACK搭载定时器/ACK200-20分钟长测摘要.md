# ACK_TIMER = 200 ms 20 分钟长测摘要

测试目的：评估 `ACK_TIMER = 200 ms` 在默认误码率和双向洪水发送压力下的长时间运行质量，并与当前默认 `ACK_TIMER = 120 ms` 的 20 分钟长测结果对比。

测试场景：默认误码率 `1.0E-5`，A/B 双站洪水发送，运行 `1200 s`。

测试命令形式：

```powershell
.\Debug\datalink.exe -f -d7 -t 1200 -p 59821 -l docs\测试记录\4.3-ACK搭载定时器\ack200-long-default-flood-20260508-205137-A.log A
.\Debug\datalink.exe -f -d7 -t 1200 -p 59821 -l docs\测试记录\4.3-ACK搭载定时器\ack200-long-default-flood-20260508-205137-B.log B
```

## 结论

`ACK_TIMER = 200 ms` 能够稳定运行 20 分钟，协议正确性没有问题，并且相比 `120 ms` 减少了独立 ACK 数量。但本次长测中 `200 ms` 的平均最后利用率低于 `120 ms`，DATA timeout 数量略高，说明减少独立 ACK 不一定能转化为更高吞吐。

当前更稳妥的选择仍是保留 `ACK_TIMER = 120 ms`。`200 ms` 可作为后续优化候选，但本次 20 分钟长测不支持直接替换当前默认值。
